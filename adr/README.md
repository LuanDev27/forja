# Decisões de arquitetura (ADRs)

Decisões que valem além de uma tarefa e cujo *motivo* precisa sobreviver a
quem as tomou. Regras não-negociáveis ficam na
[constitution](../.specify/memory/constitution.md); aqui ficam escolhas com
alternativa legítima e consequência assumida.

| # | Decisão | Status |
|---|---|---|
| [0001](0001-visao-de-integracao-ecossistema.md) | Visão de integração do ecossistema — Forja como fonte de dados Modbus, integração por Gateway externo | Aceito |
| [0002](0002-ecossistema-de-custo-zero.md) | Ecossistema de custo zero — self-host primeiro, licença permissiva por padrão | Aceito |
| [0003](0003-objetivo-e-empregabilidade.md) | O objetivo é empregabilidade, não comercialização — recalibra as armadilhas A1–A4 do 0002 | Aceito |
| [0004](0004-pick-and-place-e-o-fim-do-catalogo-congelado.md) | Pick-and-place: catálogo deixa de ser congelado e a física aprende a agarrar | Aceito |

Novo ADR: numeração sequencial, e um ADR não é editado depois de aceito —
é **substituído** por outro que o marque como `Substituído por NNNN`.
